Caso de Uso: Crear Grupo

Nombre

Crear Grupo

Descripción

Este caso de uso permite al ofertador crear un grupo para organizar y comunicar información a múltiples solicitantes. El ofertador debe escoger el tipo de grupo, ya sea privado (para invitar aplicantes) o público (para manejar solicitudes).

Actores

  • Ofertador

Evento de Activación

El caso de uso comienza cuando el ofertador decide crear un grupo asociado a una oferta de trabajo grupal existente.

Precondiciones

  • Se ha creado previamente una oferta de trabajo grupal.
  • No existe actualmente un grupo con la misma oferta de trabajo.

Flujo Principal

  1. El ofertador selecciona la oferta grupal para la cual se va a crear el grupo.
  2. El ofertador establece el nombre y la descripción del grupo, así como su tipo (público o privado).
  3. El sistema valida que el nombre y la descripción del grupo no estén en blanco.
  4. El sistema valida que se haya seleccionado el tipo de grupo.
  5. El sistema crea el grupo en la base de datos.
  6. El sistema agrega el grupo a la lista de grupos del ofertador.

Postcondiciones

Los solicitantes reciben invitaciones al grupo o notificaciones sobre la creación del grupo.

Flujos Alternativos

3a. El sistema detecta que el nombre del grupo o la descripción están en blanco

  1. El sistema indica que el nombre del grupo y la descripción no pueden estar en blanco.
  2. El ofertador confirma haber ingresado la información.
  3. El sistema crea el grupo en la base de datos con la información proporcionada.

4a. El sistema detecta que no se ha escogido el tipo de grupo

  1. El sistema indica que es necesario seleccionar el tipo de grupo.
  2. El ofertador confirma haber seleccionado la información.
  3. El sistema crea el grupo en la base de datos con la información proporcionada.
  4. El sistema agrega el grupo a la lista de grupos del ofertador.

5a. La base de datos no está disponible

  1. El sistema informa al ofertador que la base de datos no está temporalmente disponible.
  2. El ofertador confirma haber recibido la información.
  3. El sistema redirecciona al ofertador al inicio de la aplicación.
